## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (dot indicators, status dots) |
| sm | 8px | Compact element spacing (modal row padding, list item gaps) |
| md | 16px | Default element spacing (modal body padding, input spacing) |
| lg | 24px | Section padding (modal section breaks, login modal step gaps) |
| xl | 32px | Layout gaps (login modal device code field vertical centering) |
| 2xl | 48px | Major section breaks (login modal total vertical padding) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Avatar circle in TopBar: 32px diameter (8px Tailwind `w-8 h-8`) — fixed size, not a spacing token
- Sync status dot in workspace switcher: 6px diameter (`w-1.5 h-1.5`) — matches existing environment active-dot pattern in TopBar.tsx line 75
- Touch target for avatar button: 36px minimum height (HeroUI `size="sm"` Button default)

Source: Existing codebase scan — TopBar.tsx uses `h-12` (48px) bar, `px-4` (16px) padding, `gap-3` (12px). 8-point scale in effect.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.2 |

Usage notes:
- Body (14px/400): Repo rows in browser modal, avatar dropdown items, workspace switcher rows
- Label (12px/400): Visibility badge text (Private / Public), helper text below device code field
- Heading (16px/600): Modal titles ("Sign in with GitHub", "Connect a Repository")
- Display (24px/600): Device code field — the `user_code` value shown in LoginModal

Source: Existing codebase — TopBar.tsx uses `text-sm` (14px) `font-semibold` for "Dispatch" label. EnvironmentModal uses default HeroUI Modal heading. Pattern confirmed from phases 1-4.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `bg-content1` | App background, modal backgrounds, sidebar |
| Secondary (30%) | `bg-content2` / `border-divider` | Modal section dividers, repo row hover state, dropdown panels |
| Accent (10%) | `#17c964` (primary) | "Connect GitHub" button, "Connect" button per repo row, "Copy & Open GitHub" button |
| Destructive | HeroUI `danger` (#f31260) | "Sign out" in avatar dropdown (color="danger" variant="light"), disconnect confirmation "Disconnect" button |

Accent reserved for:
1. "Connect GitHub" button in TopBar (pre-login state) — `color="primary" variant="flat"`
2. "Copy & Open GitHub" button in LoginModal — `color="primary"` (filled, primary CTA)
3. "Connect" button per repo row in RepoBrowserModal — `color="primary" variant="flat"`
4. "Connected" checkmark label after successful clone — `text-primary`
5. Active workspace indicator dot in WorkspaceSwitcher — `bg-success` (green, same token as environment active dot)

Second accent — informational states:
- Sync dot green (`bg-success` = `#17c964`) for synced workspaces
- Sync dot gray (`bg-default-300`) for local-only workspaces

Source: CONTEXT.md D-09, STATE.md Phase 01-foundation decision (green primary #17c964), TopBar.tsx line 44 (`color="primary"`), existing `bg-success` usage line 75.

## Interaction States

### LoginModal (D-01, D-02)

| State | UI |
|-------|----|
| Idle (before initiate) | Not shown |
| Step 1 — Fetching code | Modal open, Spinner centered, "Requesting code..." |
| Step 2 — Awaiting approval | Device code in Display-size field (24px/600), "Copy & Open GitHub" button (primary filled), "Waiting for approval..." spinner, Cancel button (light variant) |
| Success | Modal auto-dismisses, `toast.success("Signed in as @{username}")` |
| Expired | Modal stays open, error message "Code expired. Try again." + retry button |
| Cancelled (user) | Modal closes, no toast |
| Network error | Modal stays open, error message "Could not reach GitHub. Check your connection." |

### RepoBrowserModal (D-04, D-05)

| State | UI |
|-------|----|
| Loading | Spinner centered in modal body |
| Loaded | Search input at top, repos grouped by owner name (section heading), each row: repo name + visibility icon + Connect button |
| Filtering | List filters live as user types; no empty-state needed during filtering unless zero results |
| Zero filter results | "No repos match '{query}'" in body, 14px/400, `text-default-400` |
| Cloning in progress | Row's "Connect" button replaced by inline Spinner + "Cloning..." label text; other rows remain interactive |
| Clone success | Row shows checkmark + "Connected" text (`text-primary`); workspace appears in WorkspaceSwitcher |
| Clone error | Row shows "Failed — Retry" button (`color="danger" variant="flat"`) |

### WorkspaceSwitcher (D-07, D-08, D-09)

| State | UI |
|-------|----|
| Single "Local" workspace | Dropdown trigger shows "Local", list has one entry: "Local" (no disconnect option) |
| Multiple workspaces | List shows "Local" + connected repos, each with sync dot |
| Dot: local-only | `bg-default-300` (gray) — `w-1.5 h-1.5 rounded-full` |
| Dot: synced | `bg-success` (green) — `w-1.5 h-1.5 rounded-full` |
| Bottom action | "Connect repo..." item in list, `text-primary`, triggers RepoBrowserModal |

### TopBar Auth States (D-10, D-11, D-12)

| State | UI |
|-------|----|
| Logged out | `<Button size="sm" color="primary" variant="flat">Connect GitHub</Button>` |
| Loading on startup | Button hidden, Spinner `size="sm"` in its place (brief, only during Keychain read + API verify) |
| Logged in | HeroUI `<Avatar size="sm">` with GitHub avatar URL, wrapped in Dropdown trigger |
| Avatar dropdown | Items: `{username}` (disabled display item), "Sign out" (`color="danger" variant="light"`) |
| Session expired | `toast("GitHub session expired — Sign in again", { action: { label: "Sign in", onClick: openLoginModal } })` |
