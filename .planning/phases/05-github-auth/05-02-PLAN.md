**Step 5: Create `src/stores/authStore.ts`.**

Zustand store following the same `create` pattern as collectionStore.ts:

```typescript
import { create } from 'zustand';
import * as authApi from '../api/auth';

interface GitHubUser {
  login: string;
  avatar_url: string;
  name: string | null;
  id: number;
}

interface AuthState {
  user: GitHubUser | null;
  isLoggedIn: boolean;
  isLoading: boolean;       // true during startup token check

  checkAuth: () => Promise<void>;
  setUser: (user: GitHubUser | null) => void;
  logout: () => Promise<void>;
}

export const useAuthStore = create<AuthState>((set) => ({
  user: null,
  isLoggedIn: false,
  isLoading: true,  // starts true — resolved by checkAuth on mount

  checkAuth: async () => {
    set({ isLoading: true });
    try {
      const user = await authApi.getAuthState();
      set({ user, isLoggedIn: !!user, isLoading: false });
    } catch {
      set({ user: null, isLoggedIn: false, isLoading: false });
    }
  },

  setUser: (user) => {
    set({ user, isLoggedIn: !!user });
  },

  logout: async () => {
    await authApi.logout();
    set({ user: null, isLoggedIn: false });
  },
}));
```

**Create `src/features/auth/LoginModal.tsx`.**

Props: `{ isOpen: boolean; onClose: () => void; }`.

Per D-01, D-02 (CONTEXT.md) and UI-SPEC LoginModal states:

**State machine** managed by local useState:
- `status`: `'idle' | 'fetching' | 'awaiting' | 'success' | 'expired' | 'error'`
- `deviceCode`: `DeviceCodeResponse | null`
- `errorMessage`: `string`

**HeroUI components**: Modal, ModalContent, ModalHeader, ModalBody, ModalFooter, Button, Spinner.

**On open (useEffect when isOpen becomes true and status is idle):**
1. Set status to `'fetching'`
2. Call `authApi.initiateLogin()` — on success set `deviceCode` and status to `'awaiting'`; on error set status to `'error'` with message "Could not reach GitHub. Check your connection."

**Awaiting state (useEffect when status is 'awaiting'):**
1. Call `authApi.pollLogin(deviceCode.device_code, deviceCode.interval)` — this is a long-running call that blocks until success or failure
2. On success: call `useAuthStore.getState().setUser(user)`, call `toast.success(\`Signed in as @\${user.login}\`)` (per D-02), set status to `'success'`, call `onClose()` after brief delay (100ms)
3. On error containing "expired": set status to `'expired'`, show "Code expired. Try again." with retry button
4. On error containing "denied": call `onClose()`
5. On other error: set status to `'error'`, show "Could not reach GitHub. Check your connection."

**Layout (awaiting state):**
- ModalHeader: "Sign in with GitHub" (heading 16px/600 per UI-SPEC)
- ModalBody:
  - Helper text: "Enter this code at github.com/login/device" (14px, text-default-500)
  - Device code display: `{deviceCode.user_code}` in a styled div — `text-2xl font-semibold font-mono tracking-widest text-center py-4 bg-content2 rounded-lg select-all cursor-pointer` (per UI-SPEC Display 24px/600)
  - "Copy & Open GitHub" button: `color="primary"` (filled). onClick: copy `deviceCode.user_code` to clipboard via `navigator.clipboard.writeText()`, then open `deviceCode.verification_uri` via `window.open()` (opens in default browser in Tauri).
  - Polling indicator: `<Spinner size="sm" />` + "Waiting for approval..." (14px, text-default-400)
- ModalFooter: "Cancel Sign-in" button (`variant="light"`) — calls `onClose()`. Per UI-SPEC copywriting.

**On close/unmount:** Reset status to `'idle'` and clear deviceCode. Note: the pollLogin call in Rust runs until completion; it cannot be cancelled from the frontend (fire and forget — if the modal closes, the result is discarded since the component unmounts).

**Expired state:** Show "Code expired. Try again." message + "Try Again" button that resets to 'idle' and re-triggers fetch.

**Error state:** Show errorMessage + "Try Again" button.
