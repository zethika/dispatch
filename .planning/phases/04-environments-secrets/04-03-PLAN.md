**Modify `sendRequest`:**
```typescript
sendRequest: async () => {
  const { method, url, headers, queryParams, body, auth } = get();
  const vars = useEnvironmentStore.getState().activeEnvVariables;

  // Substitute variables in all fields (per D-14, ENV-04)
  const resolvedUrl = substitute(url, vars);
  const resolvedHeaders = headers.map(h =>
    h.enabled
      ? { ...h, key: substitute(h.key, vars), value: substitute(h.value, vars) }
      : h
  );
  const resolvedQueryParams = queryParams.map(p =>
    p.enabled
      ? { ...p, key: substitute(p.key, vars), value: substitute(p.value, vars) }
      : p
  );
  const resolvedBody = body
    ? { ...body, content: substitute(body.content, vars) }
    : body;
  const resolvedAuth = auth
    ? { ...auth, token: substitute(auth.token, vars) }
    : auth;

  set({ response: { status: 'loading' } });
  try {
    const data = await httpApi.sendRequest({
      method,
      url: resolvedUrl,
      headers: resolvedHeaders,
      queryParams: resolvedQueryParams,
      body: resolvedBody,
      auth: resolvedAuth,
    });
    set({ response: { status: 'success', data } });
  } catch (e) {
    set({ response: { status: 'error', message: String(e) } });
  }
},
```

Key details:
- Only substitute in enabled entries (disabled headers/params are not sent)
- Unresolved variables remain as `{{key}}` literal text in the resolved string — the `substitute` function already handles this
- Body substitution applies to `body.content` (the JSON string). If body is null, skip.
- Auth substitution applies to `auth.token`. If auth is null, skip.
- Read `activeEnvVariables` via `useEnvironmentStore.getState()` (not a hook — this is inside a Zustand action, not a React component)

**Add the highlight overlay:**

The overlay technique: position a `<div>` with identical font, padding, and sizing on top of the `<Input>` with `pointer-events-none`. The input text is made transparent so only the caret is visible. The overlay div renders colored `<span>` tokens.

Structure inside the URL input area:
```tsx
<div className="relative flex-1">
  {/* Highlight overlay — positioned over the input, pointer-events: none */}
  <div
    aria-hidden="true"
    className="absolute inset-0 flex items-center pointer-events-none overflow-hidden whitespace-pre z-10"
    style={{ /* match HeroUI Input internal padding — verify at runtime */ }}
  >
    {tokenize(url, activeEnvVariables).map((token, i) =>
      token.type === 'variable' ? (
        <span
          key={i}
          className={token.resolved
            ? 'text-warning'  // orange/amber for resolved (D-11)
            : 'text-danger underline decoration-dotted'  // red + dotted underline for unresolved (D-12)
          }
        >
          {token.text}
        </span>
      ) : (
        <span key={i} className="text-transparent">{token.text}</span>
      )
    )}
  </div>
  {/* Actual Input with transparent text — caret remains visible */}
  <Input
    value={url}
    onValueChange={setUrl}
    classNames={{
      input: 'text-transparent caret-foreground',
    }}
    // ... existing props (size="sm", variant="bordered", placeholder, etc.)
  />
</div>
```

Important implementation details:
- The overlay div font-size, font-family, and padding must exactly match the HeroUI Input's inner `<input>` element. Inspect the rendered DOM to find the correct values. HeroUI Input with `size="sm" variant="bordered"` typically renders with `text-sm` (14px) and specific internal padding. Use the HeroUI `classNames` prop to ensure the input's internal padding matches.
- `text-warning` in HeroUI/Tailwind is an orange/amber tone — matches D-11.
- `text-danger` in HeroUI/Tailwind is red — matches D-12.
- `decoration-dotted` adds the dotted underline for unresolved variables — matches D-12.
- Literal text tokens use `text-transparent` so they are invisible — only variable tokens get color.
- The overlay scrolls with the input. For long URLs, both the input and overlay need matching `overflow-hidden` and `whitespace-pre` behavior.
