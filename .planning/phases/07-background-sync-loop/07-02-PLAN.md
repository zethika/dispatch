Add a `useEffect` to `src/App.tsx` that listens for macOS window focus changes and triggers a pull for the active GitHub-backed workspace. Per D-08, this fires once per focus transition (window gaining focus from another app), NOT on element focus within the app.

1. Add import at top of file:
```typescript

```

2. Add import for stores (if not already imported — `useWorkspaceStore` is already imported, `useSyncStore` may need import):
```typescript
import { useSyncStore } from './stores/syncStore';
```

3. Add a new `useEffect` with empty dependency array, after the existing `useEffect` hooks:
```typescript
useEffect(() => {
  let unlisten: (() => void) | undefined;
  getCurrentWindow()
    .onFocusChanged(({ payload: focused }) => {
      if (focused) {
        const { activeWorkspaceId, workspaces } = useWorkspaceStore.getState();
        const workspace = workspaces.find((w) => w.id === activeWorkspaceId);
        if (workspace && !workspace.is_local && workspace.clone_url) {
          void useSyncStore.getState().triggerPull(workspace.id);
        }
      }
    })
    .then((fn) => {
      unlisten = fn;
    });
  return () => unlisten?.();
}, []);
```

Key details:
- Uses `getCurrentWindow().onFocusChanged()` — Tauri window-level event, NOT DOM `focus` event. This fires only on macOS window focus changes (app switching), not element clicks within the WebView.
- `onFocusChanged` returns `Promise<UnlistenFn>` — the `.then()` pattern is necessary because useEffect callbacks cannot be async.
- Uses `getState()` (non-hook) to read workspace and sync store inside the callback — correct pattern for event listeners outside React render cycle.
- Checks `!workspace.is_local && workspace.clone_url` before pulling — local workspaces have no remote to pull from.
- `triggerPull` is fire-and-forget via `void` — pull result arrives via Tauri event to syncStore listener.
- Empty dependency array `[]` — listener set up once on mount, cleaned up on unmount.
