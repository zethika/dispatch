- `pub fn ensure_default_workspace(app_data_dir: &Path) -> anyhow::Result<String>` — checks if `workspaces/` has any subdirectory. If none, creates `workspaces/{uuid}/` with `dispatch.json` containing `{ id, name: "My Workspace", created_at }` and an empty `collections/` dir. Returns the workspace ID. Per D-08.
- `pub fn get_workspace_ids(app_data_dir: &Path) -> anyhow::Result<Vec<String>>` — lists subdirectories of `workspaces/`.
- `pub fn read_workspace(workspace_dir: &Path) -> anyhow::Result<WorkspaceTree>` — reads `dispatch.json`, then `collections/` subdirectories. For each collection dir with `_collection.json`, reads manifest and calls `read_children`. Silently drops order entries pointing to non-existent files (defensive read per Pitfall 2).
- `fn read_children(dir: &Path, order: &[String]) -> anyhow::Result<Vec<TreeChild>>` — iterates order array. For each slug, checks if it's a directory (folder) or `{slug}.json` file (request). Folders recurse via their `_folder.json` manifest.
- `pub fn create_collection(workspace_dir: &Path, name: &str) -> anyhow::Result<CollectionItem>` — slugify name, resolve dir collision in `collections/`, create dir, write `_collection.json` with empty order. Per D-15.
- `pub fn create_folder(parent_dir: &Path, name: &str) -> anyhow::Result<FolderItem>` — slugify, resolve dir collision, create dir, write `_folder.json`, update parent manifest order array.
- `pub fn create_request(parent_dir: &Path, name: &str) -> anyhow::Result<RequestItem>` — slugify, resolve file collision (.json), write request JSON with defaults (method: "GET", empty url, empty headers/params, null body/auth per D-13), update parent manifest order array.     io::ensure_default_workspace(&app_data).map_err(|e| e.to_string())

```typescript
import { create } from 'zustand';
import * as collectionsApi from '../api/collections';
import type { WorkspaceTree, CollectionItem, TreeChild } from '../types/collections';

interface CollectionStore {
  workspaceId: string | null;
  workspaceName: string | null;
  collections: CollectionItem[];
  expandedNodes: Set<string>;
  activeRequestId: string | null;  // slug path: "collection-slug/folder/request-slug"
  renamingNodeId: string | null;
  contextMenuNodeId: string | null;
  contextMenuPosition: { x: number; y: number } | null;

  loadWorkspace: (workspaceId: string) => Promise<void>;
  refreshWorkspace: () => Promise<void>;
  createCollection: (name: string) => Promise<void>;
  createFolder: (collectionSlug: string, parentPath: string[], name: string) => Promise<void>;
  createRequest: (collectionSlug: string, parentPath: string[], name: string) => Promise<void>;
  renameNode: (collectionSlug: string, parentPath: string[], oldSlug: string, newName: string, isDir: boolean) => Promise<void>;
  deleteNode: (collectionSlug: string, parentPath: string[], slug: string, isDir: boolean) => Promise<void>;
  duplicateRequest: (collectionSlug: string, parentPath: string[], slug: string) => Promise<void>;
  toggleExpanded: (nodeId: string) => void;
  setActiveRequest: (id: string | null) => void;
  setRenamingNode: (id: string | null) => void;
  setContextMenu: (nodeId: string | null, position: { x: number; y: number } | null) => void;
}
```

Each CRUD action (createCollection, createFolder, etc.): calls the corresponding API function, then calls `refreshWorkspace()` to reload the full tree from disk. This keeps the store in sync with disk state. `refreshWorkspace` calls `loadWorkspace` with the current `workspaceId`.

`toggleExpanded` adds/removes from `expandedNodes` Set. `setActiveRequest` sets `activeRequestId`. `setRenamingNode` sets `renamingNodeId`. `setContextMenu` sets both `contextMenuNodeId` and `contextMenuPosition`.
