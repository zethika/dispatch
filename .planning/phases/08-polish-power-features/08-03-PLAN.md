```typescript
interface SearchItem {
  nodeId: string;       // collectionSlug/parentPath.../slug

  collectionName: string;

  method?: string;
}
```

**buildSearchIndex function (export for testing):**
Walk `collectionStore.collections` recursively. For each TreeChild of type 'request', create a SearchItem with:
- `nodeId`: `collectionSlug/parentPath.join('/')/slug`
- `breadcrumb`: `CollectionName > FolderName > RequestName`
- `collectionName`: the collection's name
- `url`: the request's url field (now available in TreeChild per Task 1)
Per D-07: search scope is request names, URLs, and collection names.

**Fuse.js configuration per RESEARCH.md:**
```typescript
const fuse = useMemo(() => new Fuse(items, {
  keys: [
    { name: 'name', weight: 0.5 },
    { name: 'collectionName', weight: 0.2 },
    { name: 'url', weight: 0.3 },
  ],
  threshold: 0.4,
  includeScore: true,
  minMatchCharLength: 2,
}), [items]);
```
Rebuild the Fuse index every time the modal opens (per Pitfall 5 — prevents stale results after renames).

**Result grouping per D-08:**
Group results by `collectionSlug` into a `Map<string, { collectionName: string; items: FuseResult[] }>`.

**Result rendering per D-09 and UI-SPEC:**
- Group header: `text-xs font-semibold uppercase text-default-500`
- Result row: request name `text-sm`, breadcrumb below `text-xs text-default-400 truncate`

**Keyboard navigation:**

- Enter: call `handleSelect(selectedItem)` — calls `collectionStore.setActiveRequest(nodeId)`, expands all ancestor nodes with `collectionStore.toggleExpanded()`, closes modal
