<objective>
Build the Rust git sync engine: a thread-safe actor that serializes all git2 operations (commit, push, pull with conflict resolution), plus the IPC commands that the frontend will call.

Purpose: This is the backbone of Phase 6. The actor pattern (D-01/D-02/D-03) guarantees no concurrent git2 calls. The ops module implements the git2 API sequences for commit+push and pull-with-remote-wins-conflict-resolution (D-08/D-13). IPC commands emit Tauri events for real-time status updates (SYNC-04).

Output: `src-tauri/src/sync/` module with actor + ops + types, `src-tauri/src/commands/sync.rs` with 3 IPC commands, updated lib.rs with actor init and command registration.

**src-tauri/src/sync/types.rs** — All sync types:

2. `SyncMessage` enum (actor message protocol) with two variants:
   - `CommitAndPush { workspace_id: String, local_path: String, clone_url: String, token: String, reply: oneshot::Sender<Result<SyncResult, String>> }`
   - `Pull { workspace_id: String, local_path: String, clone_url: String, token: String, reply: oneshot::Sender<Result<SyncResult, String>> }`

3. `SyncResult` enum with `#[serde(tag = "kind", rename_all = "camelCase")]`:
   - `Pushed { had_changes: bool }`
   - `UpToDate`
   - `Pulled { conflicts: Vec<String> }` (conflict paths relative to repo root)

4. `SyncStatusPayload` struct for Tauri event emission. Fields: `workspace_id: String`, `status: String`, `message: Option<String>`, `conflicted_files: Vec<String>`. Derive `Clone, Serialize`. Use `#[serde(rename_all = "camelCase")]`.

**src-tauri/src/sync/actor.rs** — Actor loop and handle:

1. `ActorHandle` struct holding `sender: mpsc::Sender<SyncMessage>`. Derive `Clone`.

2. `ActorHandle::new()` creates mpsc channel with buffer 32, spawns `run_actor` via `tauri::async_runtime::spawn`, returns handle.

3. `ActorHandle::commit_and_push(&self, workspace_id, local_path, clone_url, token) -> Result<SyncResult, String>` — creates oneshot channel, sends `SyncMessage::CommitAndPush`, awaits reply.

4. `ActorHandle::pull(&self, workspace_id, local_path, clone_url, token) -> Result<SyncResult, String>` — same pattern with `SyncMessage::Pull`.

5. `async fn run_actor(mut receiver: mpsc::Receiver<SyncMessage>)` — loop on `receiver.recv()`. For each message variant, call `tauri::async_runtime::spawn_blocking` with the appropriate `ops::` function. Send result back via reply channel. Log errors to stderr if reply send fails.

**src-tauri/src/sync/ops.rs** — Blocking git2 functions (NEVER called from async context directly):

1. `pub fn commit_all(local_path: &str) -> Result<bool, String>`:
   - Open repo at `local_path`
   - Check `repo.statuses(None)` — iterate to find any status intersecting `INDEX_NEW | INDEX_MODIFIED | INDEX_DELETED | WT_NEW | WT_MODIFIED | WT_DELETED`
   - If no changes, return `Ok(false)`
   - Get mutable index, call `index.add_all(["*"], IndexAddOption::DEFAULT, None)`, `index.write()`
   - `index.write_tree()` to get tree OID, find tree
   - Create signature: `Signature::now("Dispatch", "sync@dispatch.app")` (per D-13)
   - Get HEAD commit as parent (handle first-commit case where HEAD doesn't exist)
   - `repo.commit(Some("HEAD"), &sig, &sig, "Dispatch sync", &tree, &parent_refs)` (per D-13)
   - Return `Ok(true)`

2. `pub fn push_to_remote(local_path: &str, clone_url: &str, token: &str) -> Result<(), String>`:
   - Open repo, find remote "origin" (or create with clone_url)
   - Set up RemoteCallbacks with `Cred::userpass_plaintext("oauth2", &token_owned)` — same pattern as clone_ops.rs
   - Get current branch from `repo.head().shorthand()` (default "main")
   - Build refspec: `format!("refs/heads/{branch}:refs/heads/{branch}")`
   - Call `remote.push(&[&refspec], Some(&mut push_opts))`

3. `pub fn pull(local_path: &str, clone_url: &str, token: &str) -> Result<Vec<String>, String>`:
   Returns Vec of conflicted file paths (empty if no conflicts).
   - Open repo, find remote "origin"
   - Fetch with credential callbacks: `remote.fetch(&["refs/heads/*:refs/remotes/origin/*"], Some(&mut fetch_opts), None)`
   - Find FETCH_HEAD: `repo.find_reference("FETCH_HEAD")`, convert to annotated commit
   - Call `repo.merge_analysis(&[&fetch_commit])`
   - If `is_up_to_date()`: return `Ok(vec![])`
   - If `is_fast_forward()`: update ref target to fetch_commit.id(), set_head, checkout_head with force. Return `Ok(vec![])`
   - Normal merge path:
     - `repo.merge(&[&fetch_commit], None, Some(&mut checkout_opts))` with force checkout
     - Get index, check `index.has_conflicts()`
     - If conflicts: iterate `index.conflicts()` collecting `their.path` into `conflicted_paths: Vec<String>` (in a scoped block so borrow drops)
     - Resolve with `checkout_index` using `use_theirs(true).force().allow_conflicts(true)`
     - `index.add_all(["*"], ...)`, `index.write()`
     - `repo.cleanup_state()` to clear MERGE_HEAD (CRITICAL — see Research pitfall 2)
   - Commit the merge: two parents (HEAD commit + fetch commit), message "Dispatch sync (merge)"
   - Return `Ok(conflicted_paths)`

4. Helper `fn get_current_branch(repo: &Repository) -> Result<String, String>` — `repo.head().shorthand()` with fallback to "main".

**src-tauri/src/commands/sync.rs** — Three IPC commands:

1. `sync_workspace(app: tauri::AppHandle, workspace_id: String) -> Result<(), String>`:
   - Use `tauri::Emitter` trait (add `use tauri::Emitter;` at top)
   - Look up workspace in registry: `registry::load_registry(&app)?.iter().find(|e| e.id == workspace_id)`
   - Validate workspace has `clone_url` (is not local) — return error "Cannot sync a local workspace"
   - Load token: `token::load_token(&app)?.ok_or_else(|| "not_authenticated".to_string())?`
   - Emit syncing status: `app.emit("sync-status-changed", SyncStatusPayload { workspace_id: workspace_id.clone(), status: "syncing".to_string(), message: None, conflicted_files: vec![] })?`
   - Get actor: `app.state::<ActorHandle>()`
   - Call `actor.commit_and_push(workspace_id.clone(), local_path.clone(), clone_url.clone(), token.clone()).await`
   - If push fails with error containing "non-fast-forward" or "rejected": attempt pull first, then retry push once (per Research open question 1)
   - Call `actor.pull(workspace_id.clone(), local_path, clone_url, token).await`
   - On success: if pull returned conflicts, emit status "synced" with `conflicted_files` populated. Otherwise emit "synced" with empty conflicts.
   - On error: emit status "error" with message field set to the error string, then return `Err(...)` with user-friendly message
   - Return `Ok(())`

2. `pull_workspace(app: tauri::AppHandle, workspace_id: String) -> Result<(), String>`:
   - Same registry lookup + token load pattern
   - Emit "syncing" status
   - Call `actor.pull(...)` only (no commit/push — this is for workspace-switch trigger per D-12)
   - On success with conflicts: emit "synced" with conflicted_files
   - On success no conflicts: emit "synced"
   - On error: emit "error"

3. `get_sync_status(app: tauri::AppHandle, workspace_id: String) -> Result<String, String>`:
   - Simple status check — look up workspace, return "local" if is_local, otherwise "synced" as default initial status (per Research open question 3)

All commands annotated with `#[tauri::command]` and `#[specta::specta]`.
